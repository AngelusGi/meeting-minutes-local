# Minute Agent

Local-first meeting minutes generator for IT consulting. Drop an OBS recording into a simple web UI and get back a full transcript plus structured, professional meeting minutes — by default without a single byte leaving your machine.

**Pipeline:** video → ffmpeg (audio extraction) → speech-to-text (local or cloud) → local LLM (minutes drafting) → `.md` + `.docx`

> The web UI and the generated minutes template are in Italian. Transcription and minutes work in Italian and English (auto-detected).

## Features

- **Local by default** — transcription runs on your machine with [faster-whisper](https://github.com/SYSTRAN/faster-whisper); NVIDIA GPUs are detected and used automatically (CUDA), with CPU int8 fallback.
- **Optional cloud transcription** — [MAI-Transcribe](https://learn.microsoft.com/azure/ai-services/speech-service/mai-transcribe) (or any model exposed by the LLM Speech API) via your own Azure AI Foundry resource.
- **Minutes drafted locally** — a structured minute (executive summary, discussion points, decisions, action items with owners, risks, next steps) is generated by a local LLM through [Ollama](https://ollama.com). No Ollama? You still get the transcript plus an empty minutes template.
- **Zero-admin setup** — a static ffmpeg binary ships with the pip dependencies; no system packages, no admin rights.
- **Non-expert friendly** — one launcher script, drag-and-drop UI, progress log, downloadable outputs.

## Outputs

Each job produces `minuta.md`, `minuta.docx`, `trascrizione.txt` and `trascrizione.srt`, downloadable from the UI and kept under `jobs/<id>/`.

## Requirements

- Python 3.10+
- [Ollama](https://ollama.com) for minutes drafting (recommended): `ollama pull llama3.1:8b`

Everything else (including ffmpeg) is installed automatically into a local virtual environment on first launch.

## Quick start

```bash
git clone https://github.com/<you>/minute-agent.git
cd minute-agent
```

- **Windows:** double-click `run.bat`
- **Linux/macOS:** `./run.sh`

The first launch creates a venv and installs dependencies; the first transcription downloads the selected Whisper model once. The browser opens at `http://127.0.0.1:8756`.

## Transcription models

| Model | Runs | Quality | Speed | Notes |
|---|---|---|---|---|
| tiny / base | local | low–fair | very fast | quick tests |
| small | local | good | ≈1× audio duration on CPU | default |
| medium | local | very good | slow on CPU | |
| large-v3 | local | best | needs NVIDIA GPU | best for mixed IT/EN audio |
| large-v3-turbo | local | high | fast on GPU | good trade-off |
| mai-transcribe-1.5 | Azure AI Foundry | best | very fast | ~$0.36/h · 300 MB limit · no diarization |
| mai-transcribe-1 | Azure AI Foundry | high | very fast | previous generation |

## Cloud transcription (Azure AI Foundry)

1. Create a **Microsoft Foundry resource for Speech** in the Azure portal.
2. In the app, select *Cloud — Azure AI Foundry*, then enter the resource name (or full endpoint) and key. The model field is free text, defaulting to `mai-transcribe-1.5`.
3. Settings persist in a local `config.json` (git-ignored — it holds your key in plain text).

Audio is uploaded as mono 16 kHz FLAC to stay within the API's 300 MB limit (~5 h of speech). Note: the LLM Speech API is in **preview** (no SLA). Minutes drafting always stays local.

## Architecture

- `app.py` — FastAPI backend: job queue, ffmpeg extraction, pluggable `TranscriptionEngine` (faster-whisper / Azure Foundry), Ollama minutes generation, Markdown→docx export.
- `static/index.html` — single-page UI, no build step.
- `run.bat` / `run.sh` — self-healing launchers (venv validation and recreation, dependency check before install).

Adding another STT provider means implementing one `TranscriptionEngine` subclass.

## Privacy

Video and extracted audio are deleted after each job; only transcript and minutes remain in `jobs/` (git-ignored). In local mode nothing leaves the machine — the only network calls are to Ollama on localhost. In cloud mode the audio is sent to **your** Azure resource only.
